# DB-16 MCPサーバーとAI駆動開発

## 概要

生成AIを業務システムに組み込む際には、LLMが複数DBに跨るデータを自然言語から取得・更新する仕組みが必要となる。本ユースケースでは、ScalarDBのMCP（Model Context Protocol）サーバーを用いて、AIアプリケーションが安全にデータ操作を行えるようにする。

## ビジネス課題

- AIエージェントが業務データにアクセスする際、不正アクセスや誤操作を防ぎたい。
- データソースが複数存在し、AIにデータ取得ロジックを実装するのが煩雑。

## システム課題

- LLMはSQLを直接実行できないため、自然言語からDB操作への変換レイヤーが必要。
- AIエージェントにDB接続情報や認証情報を埋め込むのはセキュリティリスク。

## 対象データ・環境

- CRMデータ、注文データ、マスタデータなどLLMが参照する業務データ。
- RDBMS、NoSQL、ベクトルストアなどの複数データベース。

## 導入目的・効果

- AIアプリケーションから自然言語でデータを取得・更新できるようにし、ユーザー体験を向上。
- 複数データベースに跨る操作を安全かつ統一的に実行し、AI開発の負担を軽減。

## 解決方法（ソリューション概要）

ScalarDB MCP Serverは、LLMが自然言語で指示した操作を安全にDBに反映するためのサーバーである。MCPサーバーはModel Context Protocolに準拠し、LLMからのリクエストに対してSQLやCRUD操作を生成して実行する。ScalarDBが複数DBへのトランザクションを管理するため、LLMは単一のMCPサーバー経由でデータを操作できる（参考: [scalardb.scalar-labs.com](https://scalardb.scalar-labs.com)）。

## システム構成

- LLMサービス（例：社内GPT）とMCPサーバーを接続し、自然言語の問い合わせを送信。
- MCPサーバーは問い合わせを解析し、ScalarDBを通じて複数DBにアクセスして結果を取得・更新。
- 認証や権限情報はMCPサーバーで管理し、AIエージェント側には公開しない。

## 導入メリット／ROI

- AIアプリ開発者はデータソースやトランザクション管理を意識せずに機能を実装でき、開発期間を短縮。
- セキュアに複数データベースへアクセスできるため、AIを業務シーンに安心して展開。
- データ活用の幅が広がり、自然言語インターフェースによるユーザー体験向上が期待できる。

## 類似事例・参考資料

- サポート業務や営業支援で、LLMが顧客情報や注文履歴を照会し、更新も行うユースケースが登場している。
- ScalarDB MCP ServerがLLMのデータアクセスを統一的に提供することは、ドキュメントで紹介されている（参考: [scalardb.scalar-labs.com](https://scalardb.scalar-labs.com)）。
